# ЛР1 — Сортировка выбором с интерполяционным поиском

Программа реализует **сортировку выбором** (Selection Sort) с последующим
**интерполяционным поиском** (Interpolation Search) над массивом целых чисел.
В отчёте в качестве базовой структуры рассматривается стек на односвязном списке.
Дополнительно проведена оценка вычислительной сложности алгоритмов.

## Возможности

- Выбор типа памяти: статический массив или динамический (с `realloc` при росте)
- Три способа заполнения: ручной ввод, случайная генерация, чтение из файла
- Сохранение исходного и отсортированного массива в файлы (`input.txt`, `sorted.txt`)
- Замер времени сортировки и поиска (`clock_gettime`, в миллисекундах)
- Проверка корректности сортировки (`isSorted`)

## Алгоритмы

| Алгоритм | Сложность |
|----------|-----------|
| Сортировка выбором | O(n²) |
| Интерполяционный поиск | O(log log n) в среднем, O(n) в худшем случае |

Итоговая оценка сложности программы — **O(n²)** (определяется сортировкой).

## Сборка и запуск

```bash
gcc lab1ponat.c -o lab1
./lab1
```

На Windows (MinGW):

```bash
gcc lab1ponat.c -o lab1.exe
lab1.exe
```

## Работа с программой

После запуска программа последовательно запросит:
1. Тип памяти (1 — статический, 2 — динамический)
2. Способ заполнения (1 — ручной, 2 — случайный, 3 — из файла)
3. Данные согласно выбранному способу

Затем выполнит сортировку, выведет результат и перейдёт в режим поиска
(ввод буквы вместо числа завершает работу).
